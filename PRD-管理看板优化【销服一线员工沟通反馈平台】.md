# PRD-管理看板优化【销服一线员工沟通反馈平台】

# 背景与目标

- 本次看板优化旨在通过结构化的数据呈现与全链路时效监控，实现从数据看板到明细表格的“一键下探”，缩短问题定位路径；强化反馈文案质检与多节点时效监控，提升数字化管理水平。
- 同时引入权限规范，确保“管理者看全盘、接口人看垂直”，在保障信息安全的同时，实现精准的业务闭环管理。


# 系统权限配置

- 系统需重点定义以下两类权限，系统管理员在后台“角色权限”模块，将“接口人”账号与具体的一个或多个“三级标签”进行绑定。接口人登录后，系统自动执行过滤逻辑，非其职责范围内的标签数据对其实际不可见。
| **用户角色** | **权限定义** | **数据可见性范围** |
| --- | --- | --- |
| **管理者** | 全量权限 | 可查看全国、全量分类、全量状态下的所有原声与合并需求信息，具备全局下探权限。 |
| **接口人** | 垂直权限 | 仅可见其绑定的三级标签下的数据。看板内的所有指标（总数、合并率、时效、满意度等）均需基于其所属标签自动过滤。 |

- 同时需在系统界面中整合eiam界面，实现同一页面内的权限配置

## 满意度低分警示模块

- 展示评分 ≤ 3 分的合并需求，字段包含：需求名称、三级标签、工单状态、经办人、评分及评价原文
- 支持点击跳转质检表，限制接口人权限，跳转后仅显示相应分类的需求
